# PWA App Icons

This directory should contain the application icons required for the Psycologger patient portal to be installable as a Progressive Web App on both Android and iOS devices.

## Required Icon Files

Add the following PNG files to this directory:

### Standard Icons (for Android/general PWA)
- `icon-192.png` — 192×192px app icon (launcher icon on Android, home screen)
- `icon-512.png` — 512×512px app icon (splash screen on Android)

### Maskable Icons (for adaptive icons on Android 13+)
- `icon-192-maskable.png` — 192×192px with safe zone 40px from edge (for masking)
- `icon-512-maskable.png` — 512×512px with safe zone 40px from edge (for masking)

### iOS Icon
- `apple-touch-icon-180.png` — 180×180px for iOS home screen (all corners rounded by iOS, no transparency needed)

## Design Guidelines

- **Format:** PNG with transparency (except Apple icon, which can have background)
- **Color space:** sRGB
- **Branding:** Use Psycologger's primary blue (#3b82f6) or brand colors
- **Maskable icons:** Design with important content in the center 112px circle (safe zone); the edges will be masked into rounded squares or other adaptive shapes by the OS
- **Quality:** High-quality anti-aliased graphics, no pixelation

## Tools to Generate Icons

1. **Favicon Generator** — https://realfavicongenerator.net/
   - Upload a high-res logo or design
   - Configure for PWA + iOS
   - Download all assets in one go

2. **PWA Asset Generator** — https://www.npmjs.com/package/pwa-asset-generator
   ```bash
   npm install -g pwa-asset-generator
   pwa-asset-generator logo.png ./ -b "#ffffff" -i "logo.png"
   ```
   Creates maskable variants automatically.

3. **Figma**
   - Design at 512×512 or higher
   - Export as PNG with transparency
   - Scale down to 192 using an image editor

## Current Status

Until icons are added to this directory, the web app manifest will reference icon paths that return 404. This does not break functionality, but the app will not display an icon in the home screen or browser UI. Replace these placeholders with real branded assets before deploying to production.

## iOS Considerations

- iOS uses the `<link rel="apple-touch-icon" href="/icons/apple-touch-icon-180.png" />` meta tag
- iOS **ignores** the web app manifest for icons
- iOS rounds all corners automatically (design accordingly)
- Avoid transparency on iOS icons — use a solid background or let iOS handle it

## Integration

The manifest is generated by Next.js from `src/app/manifest.ts`. It references the icon files here. The root `layout.tsx` also includes iOS-specific meta tags in the metadata export.
